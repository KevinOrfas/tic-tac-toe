Architecture Decision Records (ADRs)

This folder contains the Architecture Decision Records for the project. Each ADR documents a significant architectural choice, along with its context and consequences.

- ADR 001: Client-Server Architecture — docs/adr/001-client-server-architecture.md
- ADR 002: Routing Library Selection — docs/adr/002-routing-library-selection.md
- ADR 003: CSS Modules with BEM Naming Convention — docs/adr/003-css-modules-with-bem.md
- ADR 004: Docker Containerisation Strategy — docs/adr/004-docker-containerization.md
- ADR 005: PostgreSQL Database Selection — docs/adr/005-postgresql-database-selection.md

How to add a new ADR:
1. Create a new sequentially numbered markdown file (e.g., 005-some-decision.md).
2. Use the template: Title, Status, Context, Decision, Rationale, Consequences, Implementation, Date.
3. Add the new ADR to the list above with a short summary and link.
